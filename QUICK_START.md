# Быстрый старт

## ⚠️ Важно
Это Windows Forms приложение, которое работает **только на Windows**. Для запуска на macOS/Linux потребуется Windows машина или виртуальная машина.

## Запуск на Windows

### Вариант 1: Через Visual Studio (рекомендуется)

1. **Откройте проект:**
   - Запустите Visual Studio 2022
   - File → Open → Project/Solution
   - Выберите файл `AutoPartsStore/AutoPartsStore.csproj`

2. **Настройте базу данных:**
   - Откройте SQL Server Management Studio
   - Выполните скрипт `Database/CreateDatabase.sql`
   - (Опционально) Выполните `Database/InsertSampleData.sql`

3. **Настройте строку подключения:**
   - Откройте `AutoPartsStore/Program.cs`
   - Измените строку подключения под ваш SQL Server:
   ```csharp
   var connectionString = "Server=localhost;Database=AutoPartsStore;Integrated Security=True;TrustServerCertificate=True;";
   ```

4. **Запустите проект:**
   - Нажмите F5 или Debug → Start Debugging

### Вариант 2: Через командную строку

1. **Откройте PowerShell или Command Prompt**

2. **Перейдите в папку проекта:**
   ```cmd
   cd AutoPartsStore
   ```

3. **Восстановите пакеты:**
   ```cmd
   dotnet restore
   ```

4. **Соберите проект:**
   ```cmd
   dotnet build
   ```

5. **Запустите приложение:**
   ```cmd
   dotnet run
   ```

## Проверка работы

После запуска вы увидите форму входа. Введите учетные данные и нажмите "Войти".

### Что можно проверить:

1. **Вход как администратор:**
   - Должны быть доступны все вкладки
   - Меню "Администрирование" должно быть видно
   - Можно редактировать/удалять товары

2. **Вход как пользователь:**
   - Меню "Администрирование" скрыто
   - Кнопки редактирования/удаления товаров неактивны
   - Можно создавать заказы и продажи

## Устранение проблем

### Ошибка подключения к БД
- Убедитесь, что SQL Server запущен
- Проверьте строку подключения в `Program.cs`
- Убедитесь, что база данных создана

### Ошибка сборки
- Убедитесь, что установлен .NET 6.0 SDK или выше
- Выполните `dotnet restore`
- Проверьте, что все файлы на месте

### Приложение не запускается
- Проверьте, что используется Windows
- Убедитесь, что установлен .NET Desktop Runtime
